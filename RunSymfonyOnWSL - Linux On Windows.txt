<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>and you can run <span
class=SpellE>linux</span> commands. HOWEVER!!!<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>However:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>***Use the Ubuntu shell
instead of the windows command line as the later will throw all kinds of access
errors. It is probably better to keep <span class=SpellE>linux</span> programs
and files separate from windows programs and files. That is, make a projects
folder at <o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>linuxusername@<span
class=GramE>computername</span></span><span class=GramE>:$</span>/home/<span
class=SpellE>mylinuxusername</span> (which is the same as <span class=SpellE>linuxusername@computername</span>:$~
in Linux environment)<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>Windows files meta data and
Linux files meta data do not play well together. If you are getting permission
errors in windows command console when operating in Linux, this is probably the
problem. If you use a windows program to modify a Linux file and you can't
access or see that file in Linux, this is the problem. <o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>Some methods to avoid this
issue:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>- Generally, use Linux
programs to modify <span class=SpellE>linux</span> files and window programs to
modify windows files.<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>- Use Linux shell to
copy/move files to Linux instead of Windows command line or windows program. <o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>- If you do use a windows
program to modify a file, restart the Ubuntu shell in order to see it, try
running these commands:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
<span class=SpellE>umount</span> /<span class=SpellE>mnt</span>/c<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
mount -t <span class=SpellE>drvfs</span> C: /<span class=SpellE>mnt</span>/c -o
metadata<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>-You can use windows file
explorer to move the files. However, you won't be able to see the file
afterwards until you restart the WSL service. To restart, exit out of Ubuntu
bash, <span class=SpellE>goto</span> windows services <span class=SpellE>gui</span>
and restart <span class=SpellE>LxssManager</span>. Go back into Bash and you
should see the file.<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>- Some websites claim that
these issues only apply to files that exist in under the Linux root and that
you should save your Linux files in your Windows directory. However, I haven't
tested that.<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>################BUILD YOUR
SYMFONY ENVIRONMENT...<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>Create a file system with <span
class=SpellE>mkdir</span> so that there is a bin folder and a projects folder
under your home <span class=SpellE>dir</span>: <span class=SpellE>tlarson@LAPTOP-<span
class=GramE>DAD</span></span><span class=GramE>:~</span>$.<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>To run <span class=SpellE>symfony</span>
projects in this Ubuntu/Linux environment, <span class=GramE>You'll</span> need
to install php first, then composer.<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>Then to load php into the
Linux environment:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
add-apt-repository <span class=SpellE><span class=GramE>ppa:ondrej</span></span>/php<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
apt-get update &amp;&amp; <span class=SpellE>sudo</span> apt-get upgrade<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
apt-get install php-fpm php-<span class=SpellE>mysql</span> php7.1-cli
php7.1-curl php7.1-mbstring php7.1-mcrypt php7.1-xml php7.1-zip php7.1-intl
curl git unzip php-cli<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>(the php extensions need to <span
class=GramE>added</span> as necessary to run the <span class=SpellE>symfony</span>
environment)<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>check if php installed: php
-v<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>---------------------------------------------------------<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>To install composer:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>curl -<span class=SpellE>sS</span>
https://getcomposer.org/installer | php<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>then <o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>mv ~/<span class=SpellE><span
class=GramE>composer.phar</span></span> ~/bin/composer<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>(or you can try this
command:<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>cd ~ <o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>curl -<span class=SpellE>sS</span>
https://getcomposer.org/installer -o composer-<span class=SpellE>setup.php</span>
<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><span class=SpellE>sudo</span>
php composer-<span class=SpellE>setup.php</span> --install-<span class=SpellE>dir</span>=/<span
class=SpellE>usr</span>/local/bin --filename=composer<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>)<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>test with composer -v<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>--------------------------------------------<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>git needs to be installed so
check <span class=GramE>with:</span> git --version<o:p></o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>if it isn't, install using

<p style='margin-bottom:0in;margin-bottom:.0001pt'><o:p>&nbsp;</o:p></p>

<p style='margin-bottom:0in;margin-bottom:.0001pt'>Then git clone your project
